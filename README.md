# Sborka Mods

Репозиторий модов для лаунчера Sborka.

## Структура

```
sborka-mods/
├── mods/              ← Клиентские моды (отображаются в UI лаунчера)
│   └── *.jar
├── server-mods/       ← Серверные моды (скрыты от UI, обязательные)
│   └── *.jar
├── manifest.json      ← Манифест клиентских модов (авто-генерация)
├── server-manifest.json ← Манифест серверных модов (авто-генерация)
└── update-manifest.js ← Скрипт обновления обоих манифестов
```

## Два типа модов

### Клиентские моды (`mods/`)
- **Отображаются** в лаунчере на вкладке "Моды"
- Пользователи могут **включать/выключать** через тогглы
- Пользователи могут **удалять** через иконку корзины
- Синхронизируются из `manifest.json`

### Серверные моды (`server-mods/`)
- **НЕ отображаются** в интерфейсе лаунчера
- **Обязательные** — нельзя отключить или удалить
- Скачиваются автоматически при синхронизации
- Синхронизируются из `server-manifest.json`
- Защищены от удаления клиентской синхронизацией

## Как добавить моды

### 1. Добавить клиентский мод (видимый)
```bash
# Положить .jar файл в папку mods/
cp MyMod-1.0.jar mods/

# Обновить манифесты
node update-manifest.js

# Запушить
git add .
git commit -m "Добавлен мод MyMod"
git push
```

### 2. Добавить серверный мод (скрытый, обязательный)
```bash
# Положить .jar файл в папку server-mods/
cp ServerLib-1.0.jar server-mods/

# Обновить манифесты
node update-manifest.js

# Запушить
git add .
git commit -m "Добавлен серверный мод ServerLib"
git push
```

## Как работает синхронизация

1. При запуске лаунчера или нажатии кнопки ↻:
   - Скачиваются `server-manifest.json` и `manifest.json` с GitHub
   - Серверные моды скачиваются/обновляются (без удаления)
   - Клиентские моды скачиваются/обновляются (лишние удаляются)
   - Серверные моды защищены от удаления клиентской синхронизацией
   - Список серверных файлов сохраняется в `.server-mods.json`

2. В UI лаунчера:
   - Серверные моды отфильтровываются по `.server-mods.json`
   - Показываются только клиентские моды
